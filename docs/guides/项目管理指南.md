# 项目管理指南

> 版本: 3.1 | 最后更新: 2026-02-25
> 关联设计文档: PROJECT_MANAGEMENT / ORCHESTRATOR

---

## 一、功能概览

AutoC 支持完整的**项目隔离**和**需求级粒度控制**。每个项目拥有独立的目录、需求列表和任务追踪。

### 核心特性

- **项目隔离**: 每个项目独立目录，互不干扰
- **多项目管理**: 同一 workspace 中管理多个项目
- **需求智能优化**: 提交前自动评估需求质量
- **断点恢复**: 支持从中断处恢复执行，已通过任务自动跳过
- **CLI + Web 对等**: 命令行和 Web 界面功能完全相同

## 三、CLI 命令完整参考

### 初始化项目

```bash
autoc init my-blog -d "个人博客系统"
autoc init crm-system -t Python -t Flask -t PostgreSQL
autoc init myapp --single-task    # 单任务模式
autoc init myapp --no-git          # 禁用 Git
```

### 项目列表与状态

```bash
autoc list                         # 列出所有项目
autoc list --workspace ./workspace # 指定工作区
autoc status                       # 当前项目状态
autoc status --path ./workspace/x  # 指定项目
```

### 创建并开发

```bash
autoc run "创建 Hello World 网页"
autoc run --project-name my-blog "创建博客系统"
autoc run --project-path ./myapp "添加用户认证"
```

### 增量功能

```bash
autoc add-feature "新增评论功能"
```

### 删除项目

```bash
autoc delete my-blog                 # 删除项目及文件
autoc delete my-blog --keep-files    # 仅删除元数据
autoc delete my-blog -y              # 跳过确认
```

### 会话管理

```bash
autoc clear-sessions                 # 清除已结束会话
autoc clear-sessions --all -y        # 清除全部
```

## 四、Web API 端点完整参考

所有 API 使用 `/api/v1/` 前缀。旧版 `/api/xxx` 路径会自动 307 重定向到 `/api/v1/xxx`。

### 系统配置

```
GET    /api/v1/config                              # 系统配置
GET    /api/v1/providers                           # LLM Provider 列表
GET    /api/v1/model-config                        # 各 Agent 模型配置
PUT    /api/v1/model-config                        # 保存模型配置
POST   /api/v1/test-model                          # 测试模型连通性
GET    /api/v1/settings                            # 全局设置
PUT    /api/v1/settings                            # 更新设置
```

### 项目 CRUD

```
GET    /api/v1/projects                            # 项目列表（?detail=true 含任务详情）
GET    /api/v1/projects/{name}                     # 项目详情（含任务+会话+文件列表）
POST   /api/v1/projects                            # 新建项目
        Body: { name, folder?, description?, tech_stack?, git_enabled?, single_task?, template? }
PATCH  /api/v1/projects/{name}                     # 更新项目（名称/描述/技术栈）
        Body: { name?, description?, tech_stack?, git_enabled?, single_task? }
DELETE /api/v1/projects/{name}                     # 删除项目
        Body: { keep_files?, force? }
POST   /api/v1/projects/batch-delete               # 批量删除
        Body: { names[], keep_files?, force? }
        返回: { deleted[], skipped[], failed[] }
```

### 项目操作

```
POST   /api/v1/projects/{name}/resume              # 断点恢复（跳过 PM，已有代码直接 Test）
POST   /api/v1/projects/{name}/milestone           # 添加里程碑
        Body: { title, description?, version? }
POST   /api/v1/projects/{name}/quick-fix           # 快速修复 Bug
        Body: { bug_ids?, bug_titles?, bugs? }
POST   /api/v1/projects/{name}/revise              # 调整需求并重新执行
        Body: { requirement, clean_workspace? }
```

### 执行控制

```
POST   /api/v1/run                                  # 启动开发任务
        Body: { requirement, project_name?, tech_stack?, max_rounds?,
                max_iterations?, resume?, clean?, no_parallel?, verbose?,
                mode? ("full"|"quick"|"fix") }
        返回: { session_id }
POST   /api/v1/stop/{session_id}                   # 终止执行
```

### SSE 事件流

```
GET    /api/v1/events/{session_id}                 # SSE 实时事件流
```

### 会话管理

```
GET    /api/v1/sessions                            # 会话列表
GET    /api/v1/sessions/{id}/events                # 历史事件回放（JSON）
DELETE /api/v1/sessions                            # 清空会话
DELETE /api/v1/sessions/{id}                       # 删除单个会话
```

### 文件读写

```
GET    /api/v1/file/{session_id}?path=...          # 通过会话读取文件
GET    /api/v1/projects/{name}/file?path=...       # 通过项目名读取文件
PUT    /api/v1/projects/{name}/file                # 写入项目文件
        Body: { path, content }
```

### 需求优化

```
POST   /api/v1/refine                              # 质量评估 + 智能增强
        Body: { requirement, mode? ("enhance"|"clarify"), workspace_dir?, project_name?, tech_stack? }
POST   /api/v1/refine/merge                        # 合并澄清回答
        Body: { requirement, questions[], answers[] }
```

### AI 辅助

```
POST   /api/v1/ai-assist                           # 描述润色 / 技术栈推荐
        Body: { action ("polish"|"recommend_tech"|"both"), project_name, description? }
```

### 预览与沙箱

```
GET    /api/v1/preview/{session_id}                # 获取预览 URL
POST   /api/v1/preview/{session_id}/stop           # 停止预览服务
GET    /api/v1/sandbox/status                      # Docker 可用性 + 容器列表
POST   /api/v1/sandbox/{name}/stop                 # 停止指定沙箱容器
```

### 终端

```
WS     /api/v1/terminal/{project_name}             # WebSocket 终端（Docker 沙箱 Shell）
```

### 工具路由

```
GET    /api/v1/templates                           # 列出所有模板
POST   /api/v1/templates                           # 导出项目为模板
POST   /api/v1/import-prd                          # PRD 导入
POST   /api/v1/quick                               # 快捷模式
POST   /api/v1/fix                                 # 修复模式
POST   /api/v1/projects/{name}/deploy              # 生成部署文件
POST   /api/v1/projects/{name}/docs                # 自动生成文档
GET    /api/v1/insights                            # 历史经验洞察
```

## 五、工作流示例

### 场景 1: 创建多个独立项目

```bash
autoc run --project-name my-blog "创建个人博客，Flask + SQLite"
autoc run --project-name todo-app "创建 Todo 应用，React + Node.js"
autoc list
```

### 场景 2: 断点恢复

项目执行中断后（服务重启/手动终止），通过 Web 界面「继续执行」按钮恢复。系统自动检测已有代码，跳过 PM 规划和已通过的任务。

### 场景 3: 修复

```bash
# 定向修复 Bug
autoc fix "login.py 第 15 行的 KeyError"
```

## 六、配置说明

```yaml
workspace:
  output_dir: "./workspace"
  isolation_mode: "multi"     # multi=多项目隔离 | single=旧行为
  default_project_name: "project-{date}-{random}"
```

| 变量 | 说明 | 示例 |
|------|------|------|
| `{timestamp}` | Unix 时间戳 | `1739123456` |
| `{date}` | 日期 YYYYMMDD | `20260215` |
| `{time}` | 时间 HHMMSS | `103000` |
| `{random}` | 4位随机字符串 | `a7b3` |

## 七、Docker 沙箱

所有项目强制在 Docker 容器内执行，无法关闭。相关配置：

```yaml
features:
  sandbox_mode: "project"          # project=项目独占 | global=全局共享
  sandbox_image: "nikolaik/python-nodejs:python3.12-nodejs22" # 全栈基础镜像
```

Web 界面可在设置抽屉的「沙箱管理」中查看和停止容器。
