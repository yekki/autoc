## 二、三 Agent 模型配置推荐

### 方案一：均衡（首选）

| Agent | 推荐模型 | 理由 |
|---|---|---|
| **PM Agent** | `qwen3.5-plus` | Agent 规划设计，成本低 |
| **Developer Agent** | `glm-4.7` | 最强编码，200K 上下文 |
| **Tester Agent** | `glm-4.5` | 速度快，成本低 |

### 方案二：性能优先（GLM 全家桶）

| Agent | 推荐模型 | 备注 |
|---|---|---|
| **PM Agent** | `glm-5` | 深度推理最强 |
| **Developer Agent** | `glm-4.7` | 同方案一 |
| **Tester Agent** | `glm-4.7` | 全能型 |

### 方案三：成本优先

| Agent | 推荐模型 | 备注 |
|---|---|---|
| **PM Agent** | `glm-4.5` | 成本最低之一 |
| **Developer Agent** | `qwen3-coder-plus` | 代码专用，性价比高 |
| **Tester Agent** | `glm-4.5` | 与 PM 复用 |

## 三、选型决策树

```
额度是否充足？
├── 是 → 项目复杂度高？
│        ├── 是 → 方案二（GLM 全家桶，PM 用 glm-5）
│        └── 否 → 方案一（均衡，千问 + GLM 互补）
└── 否 → 方案三（成本优先）
```

## 四、配置方法

### 方法一：config.yaml（推荐）

在 `config.yaml` 中为每个 Agent 独立指定 `preset` 和 `model`：

```yaml
agents:
  pm:
    preset: qwen          # Provider 预设
    model: qwen3.5-plus   # 模型 ID
  developer:
    preset: glm
    model: glm-4.7
  tester:
    preset: glm
    model: glm-4.5
```

### 方法二：Web 设置面板（推荐）

打开 Web 界面 → 右上角设置 → 「模型分配」区域，为每个 Agent 选择 Provider 和模型。

配置持久化在 `autoc-models.json`，通过 Web UI 管理。

---

## 五、备用 Provider 配置

在 `autoc-models.json` 的 `fallback` 段配置，主 Provider 不可用时自动切换：

```json
{
  "fallback": {
    "pm": [
      { "provider": "qwen", "model": "qwen3.5-plus" },
      { "provider": "glm",  "model": "glm-4.7" }
    ],
    "developer": [
      { "provider": "kimi", "model": "kimi-for-coding" }
    ]
  }
}
```

按从左到右顺序尝试。401 认证错误不重试。
